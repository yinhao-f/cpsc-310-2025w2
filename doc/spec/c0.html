UniFlow (1.0.4)

Operational data management system for universities. Information is organized hierarchically: every university (ogranization) has 0 or more courses and each course has 0 or more sections.

Notes:

  • Integer fields are validated according to ECMA Script isInteger specification.

Create or replace a section for a course in an organization

If the section already exists, the entire section is replaced; otherwise it is created.

path Parameters
org
required
string
Example: ubc

Organization identifier

course
required
string
Example: cpsc310

Course identifier

section
required
string
Example: 24w201

Section identifier

Request Body schema: application/json
required
instructor
string

The instructor's full name.

year
integer^{19|20}\d{2}$

The year in which the section was offered.

avg
number [ 0 .. 100 ]

The average grade of students enroled in the section.

pass
integer >= 0

The number of students in the section who achieved a passing grade.

fail
integer >= 0

The number of students in the section who did not achieve a passing grade.

audit
integer >= 0

The number of students in the section who audited the course.

Responses
