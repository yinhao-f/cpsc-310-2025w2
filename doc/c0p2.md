<h2>Task 2: Web Frontend (30%)</h2>
<div><strong>Goal:</strong> Create a basic web frontend that interacts with your endpoints to display and edit organization, course, and section data.</div>
<div>
<h3>Requirements</h3>
<p>A user must be able to do the following things using your frontend</p>
<ol style="list-style-type: decimal;">
<li>View Organizations
<ol style="list-style-type: decimal;">
<li><span style="font-family: inherit; font-size: 1rem;">Display a list of all organizations (universities).</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Show each organization’s name.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Clicking an organization should show its list of courses.</span></li>
</ol>
</li>
<li><span style="font-family: inherit; font-size: 12pt;">View Courses</span>
<ol style="list-style-type: decimal;">
<li><span style="font-family: inherit; font-size: 1rem;">For a selected organization, display all its courses.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Show course title, department, and code.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Clicking a course should show its list of sections.</span></li>
</ol>
</li>
<li><span style="font-family: inherit; font-size: 18px;">View Sections</span>
<ol style="list-style-type: decimal;">
<li><span style="font-family: inherit; font-size: 1rem;">For a selected course, display all its sections.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Show section instructor, year, average grade, pass/fail/audit counts.</span></li>
</ol>
</li>
<li><span style="font-family: inherit; font-size: 18px;">CRUD Operations</span>
<ol style="list-style-type: decimal;">
<li><span style="font-family: inherit; font-size: 1rem;">Add a new organization, course, or section.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Edit existing organizations, courses, or sections.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">Delete organizations, courses, or sections.</span></li>
<li><span style="font-family: inherit; font-size: 1rem;">All changes should be reflected in the UI and persisted via the API.</span></li>
</ol>
</li>
<li><span style="font-family: inherit; font-size: 18px;">Usability</span>
<ol style="list-style-type: decimal;">
<li><span style="font-family: inherit; font-size: 1rem;">The UI should be clear and easy to navigate.</span></li>
</ol>
</li>
<li><span style="font-family: inherit; font-size: 18px;">Technical</span>
<ol style="list-style-type: decimal;">
<li><span style="font-family: inherit;"><span style="font-size: 1rem;">Must connect to the provided API endpoints to get and set</span></span> data.</li>
<li>Must run locally for a TA using <code>yarn demo</code> in your <code>frontend</code>&nbsp;directory.</li>
</ol>
</li>
</ol>
<h3>Developing your frontend</h3>
<ul>
<li><span style="font-family: inherit; font-size: 1rem;">Use a frontend framework (React, Vue, Svelte, etc.) or plain JavaScript.</span></li>
<li>All content (code, images) for your frontend must be contained within your frontend/ directory. In particular, do not import any code from your root src/ directory.</li>
<li>You must create a <code>package.json</code> file in your frontend directory with a script called <code>demo</code> that starts everything needed for your frontend.</li>
<li>You may add dependencies to your frontend, but <strong>do not add frontend dependencies to your backend</strong> (i.e., no code outside of your frontend directory should import a file or dependency in your frontend directory/package.json).</li>
<li>For evaluation purposes, <strong>you must preload sample data</strong> by setting the <code>DATA_DIR</code> environment variable in your <code>demo</code> script.</li>
</ul>
<p>A minimal package.json in your frontend directory would be:</p>
<div style="background: #f8f8f8; overflow: auto; width: auto; border: solid gray; border-width: .1em .1em .1em .8em; padding: .2em .6em;">
<pre style="margin: 0; line-height: 125%;">{
<span style="color: #bbb;">  </span><span style="color: #ba2121;">"scripts"</span><span style="color: #666;">:</span><span style="color: #bbb;"> </span>{
<span style="color: #bbb;">    </span><span style="color: #ba2121;">"demo"</span><span style="color: #666;">:</span><span style="color: #bbb;"> </span><span style="color: #ba2121;">"cross-env DATA_DIR=frontend/demo_data yarn --cwd .. start"</span>
<span style="color: #bbb;">  </span>}
}
</pre>
</div>
<div>The demo scripts sets DATA_DIR to a directory called demo_data in your frontend/ directory and runs the start command in the root package.json. Note that you will need to install cross-env as a frontend dependency by running <code>yarn add cross-env</code>.</div>
